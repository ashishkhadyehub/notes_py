Object-Oriented Programming (OOP) in Python

OOP (Object-Oriented Programming) is a programming paradigm that uses "objects" and "classes" to design and structure code. It helps in organizing complex programs, code reuse, and abstraction.

Concept Description
Class A blueprint for creating objects. Defines attributes and behaviors (methods).
Object An instance of a class. It holds actual data and can use the class’s methods.
__init__() Constructor method that gets called when a new object is created from the class.
self Refers to the current instance of the class. Used to access variables and methods.
Method A function defined inside a class that describes the behavior of the object.
Encapsulation Hides the internal state of an object and only exposes safe interfaces.
Inheritance Allows one class (child) to inherit attributes and methods from another class (parent).
Polymorphism Allows the same method to have different meanings based on the context (class/object).

Why Use OOP?

  • Organizes code into logical units (classes).
  • Makes code reusable (via inheritance).
  • Easier to maintain and extend.
  • Helps in building large-scale applications.

Class & Object

# Define a class
class Person:
    # Constructor
    # __ is called dunder in Python terminology 
    def __init__(self, name, age):
        self.name = name      # instance variable
        self.age = age

    # Method to display details
    def introduce(self):
        print(f"Hi, I'm {self.name} and I'm {self.age} years old.")

# Creating an object of the class
p1 = Person("Ashish", 32)

# Accessing method using the object
p1.introduce()

Inheritance

# Base class
class Car:
    def __init__(self, brand, model, fuel_type):
        # Initialize common attributes for all types of cars
        self.brand = brand
        self.model = model
        self.fuel_type = fuel_type

    def start_engine(self):
        # Method to simulate starting a fuel-based car engine
        print(f"{self.brand} {self.model}'s engine started using {self.fuel_type}.")

    def stop_engine(self):
        # Method to simulate stopping the engine
        print(f"{self.brand} {self.model}'s engine stopped.")


# Derived class (ElectricCar inherits from Car)
class ElectricCar(Car):
    def __init__(self, brand, model, battery_capacity):
        # super() is used to call the __init__ method of the parent class (Car)
        # This ensures that brand, model, and fuel_type are initialized properly
        # Even though fuel_type is not passed by the user, we hardcode it to 'Electric' for electric cars
        super().__init__(brand, model, fuel_type="Electric")

        # Initialize additional attribute specific to electric cars
        self.battery_capacity = battery_capacity

    def charge_battery(self):
        # Method specific to ElectricCar
        print(f"{self.brand} {self.model} is charging. Battery capacity: {self.battery_capacity} kWh.")

    # Overriding start_engine method from the parent class
    def start_engine(self):
        # Electric cars don't start like fuel-based cars; hence, we override the behavior
        print(f"{self.brand} {self.model} powers on silently. (Electric Start)")


# Create a Car object
my_car = Car("Audi", "A6", "Petrol")
my_car.start_engine()        # Uses Car class's method
my_car.stop_engine()

print("---")

# Create an ElectricCar object
my_electric_car = ElectricCar("Tesla", "Model 3", 75)
my_electric_car.start_engine()     # Uses overridden method in ElectricCar
my_electric_car.charge_battery()
my_electric_car.stop_engine()      # Inherited from Car (not overridden)

Muliple Inheritance

# Parent class 1
class Father:
    def gardening(self):
        print("Father enjoys gardening.")

# Parent class 2
class Mother:
    def cooking(self):
        print("Mother enjoys cooking.")

# Child class inheriting from both Father and Mother
class Child(Father, Mother):
    def sports(self):
        print("Child enjoys playing sports.")

# Create object of Child
c = Child()

# Access methods from both parent classes
c.gardening()   # Inherited from Father
c.cooking()     # Inherited from Mother
c.sports()      # Defined in Child

Encapsulation

class Employee:
    def __init__(self, name, salary):
        self.name = name               # Public attribute: can be accessed directly
        self.__salary = salary         # Private attribute: __ makes it private to the class

    def show_details(self):
        # Public method to display employee details
        print(f"Employee Name: {self.name}")
        print(f"Salary: ₹{self.__salary}")

    def set_salary(self, amount):
        # Setter method to safely update the private attribute
        if amount > 0:
            self.__salary = amount
        else:
            print("Invalid salary!")

    def get_salary(self):
        # Getter method to access the private attribute
        return self.__salary

# Creating an Employee object
emp = Employee("Shree", 50000)

# Accessing public attribute directly
print(emp.name)  # ✔ Allowed

# Trying to access private attribute directly will raise an error
# print(emp.__salary)  ❌ This will raise: AttributeError

# Instead, use getter to access the private value
print("Salary:", emp.get_salary())  # ✔ Safe access

# Using setter to update the private value
emp.set_salary(60000)

# Display updated details
emp.show_details()

Polymorphism

# Define a class named Dog
class Dog:
    # Method that returns a dog sound
    def speak(self):
        return "Play dog sound!"

# Define a class named Cat
class Cat:
    # Method that returns a cat sound
    def speak(self):
        return "Play cat sound!"


# Create an instance of Dog
d = Dog()

# Create an instance of Cat
c = Cat()

# Function demonstrating polymorphism
def play_animal_sound(animal):
    # Calls the speak() method of the object passed
    # Polymorphism in action: same method name behaves differently based on the object type
    print(animal.speak())

# Call the function with Dog object
play_animal_sound(d)  # Output: Play dog sound!

# Call the function with Cat object
play_animal_sound(c)  # Output: Play cat sound!

Static Method in class

A Static method is a method inside a class that does not depend on the instance (self) or class (cls). It's used when the function logically belongs to the class but doesn't need to access or modify class or instance data.

Static Method and Global scope variable

# Global scope variable
company_name = "Revolution"

class Employee:
    # Static method: does not access instance (self) or class (cls)
    @staticmethod
    def welcome_message():
        print("Welcome to the company!")

    # Regular method accessing global variable
    def show_company(self):
        print("Company Name (from global scope):", company_name)

# Call static method directly using class name
Employee.welcome_message()  # Output: Welcome to the company!

# Create an instance of Employee
emp1 = Employee()

# Call instance method that uses global variable
emp1.show_company()  # Output: Company Name (from global scope): Revolution
